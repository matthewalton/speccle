# 0014 — A skill bundles the docs it orders you to read

- Status: accepted (citations amended by
  [ADR-0028](0028-shipped-skills-carry-no-repo-citations.md))
- Date: 2026-07-09

## Context

Publishing the plugin ([ADR-0001](0001-plugin-first-tools-serve-skills.md)) revealed that
Claude Code caches only the plugin's own source directory, so the skills' `../../../../`
doc links were dead on install. Replacing them with `github.com` URLs fixed the links but
not the problem underneath: two of those links were not citations, they were
**instructions**. `implement-feature` said "read the convention before drafting";
`strengthen` said "read ADR-0012 before routing anything".

An instruction to read a URL is an instruction to `WebFetch`. A skill cannot promise that
will work — offline and sandboxed sessions have no network — so the skill was issuing an
order it could not guarantee was executable, and had no way to notice when it was skipped.
Bundling was considered and rejected on the grounds that copies drift from their source.

Rereading the two orders narrowed the question. `strengthen`'s §4 already restates
ADR-0012's decision in full; the skill explained how to route and _then_ told you to go
read how to route. Only `implement-feature`'s convention link was load-bearing — the
frontmatter and heading formats it names are not restated anywhere in the skill, and the
skill forbids hand-checking the convention in the oracle's place. One file, not four.

## Decision

A skill may only **order** the agent to read a doc that is bundled beside it, under
`skills/<name>/references/`. Anything reachable only over the network is a citation —
cite it, never instruct the agent to fetch it.

`docs/convention.md` is bundled into `implement-feature` on that rule. The copy is
generated by `scripts/sync-plugin-references.mjs`, which rewrites the source's relative
links to absolute `github.com` URLs, and `pnpm check:plugin-refs` fails the pre-commit
hook when the copy drifts. Editing the copy by hand is a mistake the check catches.

Every link that leaves `packages/plugin` is an absolute URL, for the same reason the
relative ones died: an installed plugin has no parent directory.

## Consequences

- A skill's instructions are executable offline. Whether an agent _obeys_ them is a
  separate question, and an open one
  (`strengthen` has never been run cold by an agent).
- Drift is a build failure rather than a silent divergence, which is the objection to
  bundling answered rather than accepted.
- The bundle stays at one file. A skill that wants a second must first show the doc is
  genuinely operative and not already restated in its own body — the ADR-0012 case is the
  precedent for demoting rather than bundling.
- Provenance links keep pointing at `main`, so a reader on plugin 0.1.0 may follow a
  citation to a doc that has since changed. Acceptable for prose nobody is ordered to
  read; it would not be for a bundled reference, which is why those are copied.
